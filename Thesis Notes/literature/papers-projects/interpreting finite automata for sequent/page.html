Aspects of Interpretability in Automata
1. Graphical representation
  (cyclic, directed, labeled)
2. Transparent computation
  Each step of computation is the same for each symbol wi of an input sample w. It is possible to analyze training samples.
3. Generative automate
  Sampling from models helps to understand what it describes. Model checkers query properties in a formal way
4. Well studied automata (theory and practice)
  composition, closure properties, sub-classes, etc. makes it easy to transfer knowledge to the domain. Model is often used to describe system logic and is accessible to a wide audience.

Summary
This paper identifies key properties used to interpret automata and proposes a modified state-merging approach to learn variants of finite state automata. Interpretation of state automata is not feature inherent in the models or the algorithm, but is defined in the need and intention of the user. Interpretations draw from the set of properties: graphical representation, transparent computation, generative nature and the human understanding of automata theory. Machine learning serves as a tool for exploration to deal with epistemic uncertainty in observed ststems. The goal is not only to obtain a more compact view, but also to learn how to generalize from the observed data. Flexible state merging is introduced that seperates the symbolic representation from the objective function and heuristic, which helps stating the parameters
